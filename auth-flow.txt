Логика авторизации на фронтенде

1. Основные компоненты:
- AuthContext (контекст для управления состоянием авторизации)
- LoginForm (форма входа)
- PrivateRoute (защищенный маршрут)
- apiClient (HTTP-клиент с перехватчиками)

2. Процесс авторизации:
- Пользователь вводит учетные данные в LoginForm
- Форма отправляет POST-запрос на /auth/login
- При успешной авторизации:
  * Токен сохраняется в localStorage
  * Состояние авторизации обновляется в AuthContext
  * Происходит редирект на главную страницу
- При ошибке показывается уведомление

3. Защита маршрутов:
- PrivateRoute проверяет наличие пользователя
- Если пользователь не авторизован - редирект на /login
- При загрузке показывается состояние загрузки

4. Работа с токеном:
- Токен хранится в localStorage
- apiClient автоматически добавляет токен в заголовки
- При получении 401 ошибки:
  * Токен удаляется
  * Происходит редирект на страницу входа

5. Выход из системы:
- Удаление токена из localStorage
- Сброс состояния в AuthContext
- Редирект на страницу входа

Текущие ограничения:
- Нет обновления токена
- Отсутствует запоминание последней страницы перед редиректом на логин
- Нет проверки срока действия токена
